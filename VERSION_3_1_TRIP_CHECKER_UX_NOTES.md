# Version 3.1 — Persistent Trip Checker UX

This update preserves a user's trip selections and prevents the full-rule workflow from forcing them to start again.

## Included

- Local browser persistence for airline, destination, baggage mode and selected items.
- Automatic restoration after refresh or reopening the checker.
- Full-rule links open in a new tab.
- Rule pages show a Return to Trip Checker banner when a saved trip exists.
- Dynamic Trip Readiness score.
- Start New Trip control with confirmation.
- Copy, native share and print actions.
- Cleaner print output focused on the trip summary.
- Build-time Version 3.1 workflow validation.
